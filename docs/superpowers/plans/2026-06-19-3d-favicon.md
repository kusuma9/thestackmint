# 3D Isometric Cube Favicon Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the inline SVG data-URI favicon with a proper `public/favicon.svg` file containing a 3D isometric green cube.

**Architecture:** Create one new SVG file and update one `<link>` tag. No build changes, no dependencies — Astro serves `public/` files as-is, so `favicon.svg` is available at `/favicon.svg` in both dev and production.

**Tech Stack:** SVG, Astro 4, Cloudflare Pages

## Global Constraints

- SVG viewBox must be `0 0 100 100`
- Face fills: top `#c8e6ce`, left `#1b6b3a`, right `#0f3d1f`
- Edge stroke: `#114a27`, width `0.5`
- Transparent background (no `<rect>` fill)
- Favicon `<link>` must use `type="image/svg+xml"` and `href="/favicon.svg"`

### Task 1: Create `public/favicon.svg`

**Files:**
- Create: `public/favicon.svg`

**Interfaces:**
- Produces: `/favicon.svg` served statically — consumed by the `<link>` tag in Task 2

- [ ] **Step 1: Create the SVG file**

Create `public/favicon.svg` with this exact content:

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 100 100">
  <!-- Top face (lit) -->
  <polygon points="50,10 86,30 50,50 14,30" fill="#c8e6ce" stroke="#114a27" stroke-width="0.5" stroke-linejoin="round"/>
  <!-- Left face (brand green) -->
  <polygon points="14,30 50,50 50,90 14,70" fill="#1b6b3a" stroke="#114a27" stroke-width="0.5" stroke-linejoin="round"/>
  <!-- Right face (shadow) -->
  <polygon points="50,50 86,30 86,70 50,90" fill="#0f3d1f" stroke="#114a27" stroke-width="0.5" stroke-linejoin="round"/>
</svg>
```

- [ ] **Step 2: Verify the file renders correctly**

Open `public/favicon.svg` directly in a browser. You should see:
- An isometric cube occupying the full viewport
- Three clearly distinct faces: light mint top, medium green left, very dark green right
- Thin dark edges between faces

- [ ] **Step 3: Commit**

```bash
git add public/favicon.svg
git commit -m "feat: add 3D isometric cube favicon SVG"
```

### Task 2: Wire favicon in BaseLayout.astro

**Files:**
- Modify: `src/layouts/BaseLayout.astro:44-47`

**Interfaces:**
- Consumes: `/favicon.svg` produced by Task 1

- [ ] **Step 1: Replace the inline favicon link**

In `src/layouts/BaseLayout.astro`, find lines 44–47 (the `<!-- Favicon -->` block):

```astro
    <!-- Favicon -->
    <link
      rel="icon"
      href="data:image/svg+xml,<svg xmlns='http://www.w3.org/2000/svg' viewBox='0 0 100 100'><rect width='100' height='100' rx='22' fill='%231B6B3A'/><text x='50' y='74' font-size='64' text-anchor='middle' font-family='serif' font-weight='900' fill='white'>T</text></svg>"
    />
```

Replace with:

```astro
    <!-- Favicon -->
    <link rel="icon" type="image/svg+xml" href="/favicon.svg" />
```

- [ ] **Step 2: Start dev server and verify**

```bash
npm run dev
```

Open `http://localhost:4321` in a browser. Check the browser tab — you should see the isometric green cube instead of the green "T". Also check in multiple tabs to confirm it shows consistently.

- [ ] **Step 3: Commit**

```bash
git add src/layouts/BaseLayout.astro
git commit -m "feat: wire 3D isometric cube favicon in BaseLayout"
```
